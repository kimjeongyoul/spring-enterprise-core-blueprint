# Git Commit Conventions

> **본 프로젝트의 깨끗한 히스토리 관리와 코드 가독성을 위한 커밋 규격**

## 📝 커밋 메시지 구조
```text
type: description (한글 또는 영문)

[Optional Body]
```

## 🏷 Type 종류
*   **feat:** 새로운 기능 추가
*   **fix:** 버그 수정
*   **docs:** 문서 수정 (README, ENGINEERING_VALUE 등)
*   **style:** 코드 포맷팅, 세미콜론 누락 등 (코드 변경 없음)
*   **refactor:** 코드 리팩토링 (기능 변화 없음)
*   **test:** 테스트 코드 추가 및 수정
*   **chore:** 빌드 업무 수정, 패키지 매니저 설정 등
*   **rename:** 파일 혹은 폴더명 수정 및 이동

## 💡 작성 규칙
1.  **제목(Subject):** 50자 이내로 명확하게 작성합니다.
2.  **본문(Body):** 선택 사항이며, '무엇을' 보다는 '왜' 변경했는지를 설명합니다.
3.  **변경 단위:** 하나의 커밋에는 하나의 변경 사항만 담습니다. (여러 기능을 섞지 않음)

---
**Example:** `feat: JWT 기반의 사용자 인증 필터 및 Provider 구현`
